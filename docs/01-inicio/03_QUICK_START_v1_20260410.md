# 🚀 Guía Rápida de Inicio - NEO DMSTK

Esta guía te ayudará a poner la app en funcionamiento en **15 minutos**.

## Paso 1: Configurar Supabase (5 min)

1. Ve a [supabase.com](https://supabase.com) → Sign Up (gratis)
2. Crea un nuevo proyecto:
   - Nombre: `neo-dmstk`
   - Base de datos: Elige una contraseña segura
   - Región: Elige la más cercana (EU West si estás en España)
3. Espera ~2 minutos mientras Supabase crea tu base de datos

4. Ve a **SQL Editor** (menú lateral) → **New Query**
5. Copia todo el contenido de `backend/supabase-schema.sql`
6. Pega y ejecuta con **RUN** o **Ctrl+Enter**
7. Deberías ver: ✅ Success

8. Ve a **Settings → API**:
   - Copia `Project URL` (algo como: `https://abcdefg.supabase.co`)
   - Copia `anon public` key (larga cadena de caracteres)
   - Copia `service_role` key (⚠️ NUNCA la compartas públicamente)

## Paso 2: Configurar el Backend (3 min)

```bash
cd backend
npm install
cp .env.example .env
nano .env  # o usa tu editor favorito
```

Pega tus credenciales en `.env`:
```env
SUPABASE_URL=https://tu-proyecto.supabase.co
SUPABASE_ANON_KEY=pega-aqui-tu-anon-key
SUPABASE_SERVICE_KEY=pega-aqui-tu-service-key
OPENAI_API_KEY=  # Déjalo vacío por ahora
PORT=3001
FRONTEND_URL=http://localhost:5173
```

Guarda el archivo.

## Paso 3: Configurar el Frontend (2 min)

```bash
cd ../frontend
npm install
cp .env.example .env
nano .env
```

Pega en `.env`:
```env
VITE_SUPABASE_URL=https://tu-proyecto.supabase.co
VITE_SUPABASE_ANON_KEY=pega-aqui-tu-anon-key
VITE_BACKEND_URL=http://localhost:3001
```

Guarda el archivo.

## Paso 4: Migrar tus datos (2 min)

```bash
cd ../backend
node migrate-data.js
```

Deberías ver:
```
🔄 Iniciando migración de datos...
📝 Insertando tareas...
✅ Migración completada exitosamente!
📊 Tareas insertadas
```

## Paso 5: ¡Arrancar! (1 min)

Abre **DOS terminales**:

**Terminal 1 - Backend:**
```bash
cd backend
npm run dev
```

Deberías ver:
```
🚀 Servidor corriendo en puerto 3001
📊 Dashboard: http://localhost:3001
🤖 Agente de monitoreo activo
```

**Terminal 2 - Frontend:**
```bash
cd frontend
npm run dev
```

Deberías ver:
```
VITE ready in 500 ms
➜  Local:   http://localhost:5173/
```

## Paso 6: ¡Pruébalo!

1. Abre [http://localhost:5173](http://localhost:5173)
2. Deberías ver tu dashboard con todas las tareas
3. Haz clic en el **icono de chat** (esquina inferior derecha)
4. Pregunta: "¿Cuántas tareas están vencidas?"
5. Abre la app en otra pestaña → Los cambios se sincronizan automáticamente ✨

## ❓ Solución de problemas

### "Cannot connect to server"
- Verifica que el backend esté corriendo en puerto 3001
- Revisa `backend/.env` → las URLs de Supabase son correctas

### "Tareas no aparecen"
- Ve a Supabase → Table Editor → tasks
- ¿Ves las tareas ahí? Si no, ejecuta `node migrate-data.js` de nuevo

### "Socket.IO desconectado"
- Verifica que ambos (frontend y backend) estén corriendo
- Limpia caché del navegador (Ctrl+Shift+R)

### Puerto 3001 ocupado
```bash
lsof -i :3001  # Ver qué está usando el puerto
# Cambiar PORT=3002 en backend/.env
```
