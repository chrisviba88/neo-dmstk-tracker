CAUSA RAÍZ IDENTIFICADA:
━━━━━━━━━━━━━━━━━━━━━━━
Existe un TRIGGER SQL AUTOMÁTICO en Supabase que se ejecuta cada vez que
guardas una tarea, intentando insertar en una columna 'changes' que NO existe.

Este trigger está definido en supabase-schema.sql (líneas 103-105):
  CREATE TRIGGER log_task_changes_trigger
  AFTER INSERT OR UPDATE OR DELETE ON tasks
  FOR EACH ROW EXECUTE FUNCTION log_task_changes();

POR ESO el error persiste incluso después de comentar logChange() en Node.js.

SOLUCIÓN (2 MINUTOS):
━━━━━━━━━━━━━━━━━━━━

1. Ve a: https://supabase.com/dashboard
2. Selecciona tu proyecto Neo DMSTK
3. Ve a: SQL Editor → New query
4. Copia y pega esto:

   DROP TRIGGER IF EXISTS log_task_changes_trigger ON tasks;
   DROP FUNCTION IF EXISTS log_task_changes();

5. Haz clic en RUN
6. ¡LISTO! El guardado ahora funcionará.
