# 📊 ESTRUCTURA DE PROYECTOS - ORGANIZACIÓN VISUAL

## 🎯 FILOSOFÍA DE ORGANIZACIÓN

El sistema usa una **jerarquía de 2 niveles** para organizar las 227 tareas:

```
PROYECTO PADRE > SUBPROYECTO
```

Esto permite:
- ✅ Agrupar por proyecto padre (vista macro)
- ✅ Expandir/colapsar subproyectos (vista detalle)
- ✅ Filtrar por cualquier nivel
- ✅ Visualización tipo Monday.com profesional

---

Todas las tareas que son **prerequisito global** para abrir cualquier espacio.


**Ejemplos:**

## 📱 VISTAS CONFIGURADAS

### Vista 1: "Roadmap Maestro por Proyecto"

**Agrupar:** Proyecto Padre
**Expandir:** Subproyectos automáticamente
**Ordenar:** Fecha inicio
**Mostrar:** Iconos + Colores + Progress bar

```
📂 Fundación & Método (27) [████████░░] 80%
📂 Piloto & Validación (17) [███░░░░░░░] 30%
📂 Espacio E1 — Madrid (50) [█████░░░░░] 50%
  ├─ Legal & Licencias [███████░░░] 70%
  ├─ Daruma 3D [██░░░░░░░░] 20%
  └─ ...
```

### Vista 2: "E1 Madrid - Vista Detallada"

**Filtrar:** Proyecto LIKE "%Espacio E1%"
**Agrupar:** Subproyecto
**Ordenar:** Prioridad > Fecha
**Destacar:** Tareas críticas en rojo

### Vista 3: "Critical Path"

**Filtrar:** isMilestone = Sí OR Prioridad = P0
**Mostrar:** Timeline Gantt con dependencias
**Colores:** Rojo (urgente), Amarillo (próxima semana), Verde (ok)
